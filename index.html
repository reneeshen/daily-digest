Agent 進駐設計檔，使用者開始反彈

Agents Move Into Design Files, Users Push Back

這週兩個方向撞一起：一邊 Figma 把 MCP 打開讓 AI agent 直接改檔案、Google Stitch 導入 DESIGN.md 格式，design-to-code 的管線第一次真正由 agent 接管。

另一邊 NN/g 連發兩篇提醒——使用者已經「AI 疲勞」，想看得出是人做的設計；而 agent 本身又成了介面的第二種用戶。

設計師今年的新技能會是「寫 instruction 給 agent」與「設計給 agent 看的 UI」——這兩件事，比學哪個 AI 工具更關鍵。

Two currents collided this week: Figma opened its MCP so AI agents can modify files directly, and Google Stitch rolled out DESIGN.md — the design-to-code pipeline is now genuinely agent-driven.

Meanwhile NN/g published twice this week on the opposite signal: users are AI-fatigued and want designs that look human-made, while agents themselves have become a second type of user for interfaces.

The new skill for designers this year is writing instructions for agents and designing UIs agents can read — both matter more than picking which AI tool to learn.

Google Stitch 丟出 DESIGN.md，Figma 股價一天跌 4%

Google Stitch Drops DESIGN.md, Figma Stock Falls 4%

Google Stitch 導入 DESIGN.md——用自然語言檔案記錄介面設計，可從任何 URL 抽取並跨專案重用。新增 Gemini 2.5 Pro / Flash 雙模型選擇（品質 vs. 速度），並支援 design token 上傳。市場反應強烈：3 月 19 日 Figma 股價下跌超過 4%。

Google Stitch introduced DESIGN.md — a natural-language file format that captures interface design details, extractable from any URL and reusable across projects. New model picker lets users choose Gemini 2.5 Pro for polished output or 2.5 Flash for fast iteration. Design system import now supports color palettes, typography scales, and component libraries. Market signal was immediate: Figma shares dropped over 4% on March 19.

DESIGN.md 要是變成設計界的 OpenAPI，遊戲規則就變了。但 Stitch 還在穩定性問題（論壇 10K+ views 的 bug 討論），現在還是早期試用期。

If DESIGN.md becomes the OpenAPI of design, the rules of the game change. Stitch still has stability issues (10K+ views on bug threads), so treat this as an early-access bet, not a replacement.

AI Agent 也是你的使用者

AI Agents Are Now a Second User

NN/g 指出介面現在同時服務兩種使用者——人類與 AI agent。設計時要同時思考兩者的資訊架構與無障礙：agent 看得懂 DOM 語意、有正確的 ARIA role、能被程式化操作，才能真正參與工作流。「user」的定義要重新想。

NN/g argues interfaces now serve two user types — humans and AI agents. Designers must think about information architecture and accessibility for both: agents need clear DOM semantics, correct ARIA roles, and programmatic affordances to actually participate in workflows. The very definition of "user" needs a rethink.

Accessibility 以前是 nice-to-have，現在是 agent-readable 的前提。這篇可以當下個 design review 的 checklist 起點。

Accessibility used to be nice-to-have; now it's the precondition for agent-readability. Treat this as the starting checklist for your next design review.
